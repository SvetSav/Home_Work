# Marketplace - Система управления продуктами и категориями

## Описание проекта
Проект представляет собой систему управления продуктами и категориями для интернет-магазина.

## Функциональность

### 2. Классы продуктов

#### Product (Базовый продукт)
- `name` - название
- `description` - описание  
- `price` - цена (с валидацией)
- `quantity` - количество

#### Smartphone (Смартфон) - наследник Product
- `efficiency` - производительность
- `model` - модель
- `memory` - объем встроенной памяти (ГБ)
- `color` - цвет

#### LawnGrass (Газонная трава) - наследник Product
- `country` - страна-производитель
- `germination_period` - срок прорастания
- `color` - цвет

### 3. Класс Category (Категория)
- Управление списком продуктов
- Подсчет количества категорий и продуктов
- Итератор для перебора товаров
- Валидация типов при добавлении продуктов

### 4. Класс Order (Заказ) - дополнительное задание
- Абстрактный класс AbstractBase для общих свойств
- Один товар в заказе
- Количество и итоговая стоимость
- Наследование от AbstractBase

### 5. Ограничения и валидация
- **Сложение продуктов** - только для объектов одного класса (через `type()`)
- **Добавление в категорию** - только экземпляры Product и его наследников (через `isinstance()`)
- **Цена продукта** - не может быть отрицательной или нулевой

### Дополнительная:
3. **Загрузка из JSON** - функция `load_categories_from_json()` для загрузки данных из JSON файла.

## Структура проекта
- `main.py` - основной файл с реализацией классов и демонстрацией работы
- `products.json` - JSON файл с данными о категориях и продуктах
- `tests/` - директория с тестами
- `pyproject.toml` - конфигурация Poetry и зависимостей

## Запуск демонстрации

python main.py

Тесты:

poetry run pytest tests/ -v * Все тесты

poetry run coverage run --source=. -m pytest tests/ * Все тесты с покрытием

poetry run python -m pytest tests/ --cov=. --cov-report=term * С отчетом о покрытии
